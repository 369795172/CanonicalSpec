标题: CI/CD ChatOps 通知 + 自动 PR

背景:
当前 CI/CD 已具备 dev 部署与 prd 双机发布，但仍依赖人工创建 PR 且缺少飞书自动通知链接，导致上线链路可视化与操作闭环不足。

目标:
在 dev 部署成功后自动创建 dev-backend -> prd-backend 的 PR，并在飞书通知中附带 PR 链接；在 prd 部署完成或失败时发送飞书通知，包含提交/镜像/运行信息，确保发布过程可追踪。

验收标准:
- AC-1: dev 部署成功后自动创建或复用一个指向 prd-backend 的 PR，标题包含 [Cursor]，PR 描述包含镜像 tag 与提交信息。
- AC-2: dev 部署成功后飞书卡片通知包含 PR 链接、提交链接与工作流链接。
- AC-3: prd 部署完成或失败均发送飞书通知，包含部署目标与 CVM 状态。
